# Context Base

This is the knowledge layer that grounds the AI. The models everyone uses are
the same; the only durable edge is the context you feed them that nobody else
has. Generic input produces generic output, because "average" is the safest
guess when the model knows nothing about you. These five docs are what move the
output from average to specifically-yours.

## The files

| File | What it answers | Used when the AI is… |
|------|-----------------|----------------------|
| `01-offer.md` | What you sell, to whom, the outcome, the proof | …writing anything that pitches |
| `02-icp.md` | Exactly who you're targeting and the pain they feel now | …filtering lists, writing openers |
| `03-positioning.md` | Why you, not the alternatives | …handling objections, differentiating |
| `04-proof.md` | Concrete results, numbers, names you can cite | …making any claim credible |
| `05-voice-profile.md` | How you sound, and what you'd never say | …writing in first person as you |

## How to use them

1. **Fill them in for real.** Every `> FILL IN` block is a prompt. Replace it
   with specifics. Placeholders left in = slop out. If you can't fill a section
   honestly yet, that's a signal the underlying business question isn't answered
   — fix that first, don't paper over it.
2. **Load them as context** at the start of any AI session. Paste the relevant
   docs, or attach them, before you ask for content or outreach.
3. **Assign a role.** "You are my cold outreach specialist. Here is my context
   base. Write…" beats a bare prompt every time.
4. **Keep them current.** When proof changes, when the offer sharpens, when a
   message angle starts converting — update the doc. The feedback loop lives
   here: `04-proof.md` and `05-voice-profile.md` should grow every week from
   what actually worked.

## The quality bar

Each doc contains one worked **EXAMPLE** block (fictional — a B2B agency that
fixes SaaS onboarding) so you can see the level of specificity required. Yours
should be at least that concrete. If a competitor read your doc, they should
*not* be able to use it for their own business. If they could, it's too generic.

## Honest limit

These docs make the AI's drafts good. They don't make them final. You still
edit and direct — you're the director, not a passenger. The context raises the
floor; your judgment sets the ceiling.
